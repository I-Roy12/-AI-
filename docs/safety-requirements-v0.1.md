# 体調日記AIアプリ 安全要件 v0.1

## 1. 安全方針
- AIは医療診断を行わない
- AIは治療方針を断定しない
- AIは受診行動を補助する
- 緊急時は通常機能より安全導線を優先する

## 2. AIの許可/禁止
### 許可
- 記録内容の要約
- 過去比較による傾向表示
- 症状カテゴリに基づく受診先候補提示
- 受診相談の提案

### 禁止
- 病名の断定
- 処方/中止の指示（例: 薬をやめるべき）
- 「問題ない」と断定して受診を妨げる表現
- 緊急サインがあるのに通常提案を続ける挙動

## 3. 緊急サイン検知ルール（MVP）
### 危険ワード例
- 胸が強く痛い
- 息が苦しい
- 意識が遠のく
- 死にたい / 消えたい
- 出血が止まらない

### 検知時の必須挙動
- 通常応答停止
- 緊急ガイダンスを最優先表示
- 医療機関/相談窓口への連絡導線表示
- ログ保存（時刻、検知ワード、表示した案内）

## 4. 固定文言（MVP）
### 常時表示
- 「本アプリの提案は自己記録に基づく補助情報であり、診断ではありません。」

### 緊急時表示
- 「緊急性のある可能性があります。すぐに医療機関または緊急窓口へ連絡してください。」

## 5. 出力制御
- 診断を想起させる語（例: 断定病名）はフィルタ
- 提案文は「可能性」「傾向」「相談を検討」形式に限定
- 確信度を低/中/高で併記
- 根拠データ（日時・項目）を明示

## 6. 共有と同意
- 医師共有はユーザーの明示操作時のみ
- 自動共有はデフォルト無効
- 共有時は対象データ範囲を確認画面で再提示

## 7. 監査ログ（最低限）
- AI生成日時
- 入力データ範囲
- 出力内容
- 安全フィルタ適用有無
- 緊急モード発火有無

## 8. テスト観点（MVP）
- 緊急ワード入力時に通常提案が出ない
- 禁止表現が出力されない
- 常時免責文が全画面で表示される
- 共有が明示操作なしで実行されない

## 9. 未確定事項（次回決定）
- 危険ワード辞書の拡張範囲
- 緊急窓口の地域別出し分け
- 医師候補データの検証方法
