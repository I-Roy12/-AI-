# 体調日記AIアプリ MVP定義 v0.1

## 1. 目的
- ユーザーが毎日30秒程度で体調を記録できること
- AIが記録を要約し、症状カテゴリに基づいて受診先候補を提案できること
- 緊急サイン検知時に、通常提案より先に安全導線を出せること

## 2. MVPスコープ
### 含める
- 体調入力（タップ + スライド）
- 日次サマリー（AI要約）
- 直近7日トレンド表示
- 受診先候補マッチング（症状カテゴリベース）
- 緊急サイン検知と固定アラート表示

### 含めない（MVP外）
- 病名の推定表示
- 診断や治療指示
- 医療機関への自動送信
- 決済、保険請求、電子カルテ連携

## 3. 入力項目（毎日）
- 記録日時（自動）
- 主症状（複数選択）
- 症状強度（0-10 スライダー）
- 気分（0-10 スライダー）
- 睡眠時間（時間）
- 睡眠の質（0-10 スライダー）
- 服薬（飲んだ / 飲み忘れ）
- 自由記述（任意）

## 4. AI出力（MVP）
- 今日の要約（2-3行）
- 7日傾向（悪化/横ばい/改善）
- 受診先候補（診療科 + 医師候補）
- 根拠表示（どの記録を見て提案したか）
- 確信度表示（低/中/高）

## 5. マッチングルール（簡易）
- 症状カテゴリ -> 診療科候補へマッピング
- 条件: 対応症状、オンライン可否、予約可能時間、説明スタイル
- 並び順: 適合度スコア -> 直近予約可能時間
- 重要: 「診断ではなく受診先選びの補助」であることを常時表示

## 6. 緊急時の挙動
- 危険ワード検知時:
  - 通常のAI要約/マッチングは停止
  - 緊急案内画面を最優先表示
  - 緊急相談窓口・受診導線を表示

## 7. 受け入れ基準（MVP完了条件）
- 入力画面で30秒以内に記録完了できる
- AI要約が毎回生成される
- 少なくとも1件以上の受診先候補を返せる
- 危険ワード入力で通常画面に遷移せず、緊急画面が表示される
