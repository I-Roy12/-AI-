# 公開前リーガルチェックリスト v1（体調日記）

最終更新: 2026-03-10  
対象: 体調日記アプリ（患者記録 + AI要約 + 医師共有）

本書は法的助言そのものではなく、公開前に法務・セキュリティ担当と確認するための実務チェックリストです。

## 0. 現状判定（このまま公開してよいか）

- 判定: 一般公開は非推奨（デモ限定なら可）
- 理由:
  - 健康情報（要配慮個人情報）を扱う設計で、同意・第三者提供説明・運用体制が不足
  - `u_123` ベースの利用が可能で、利用者分離の前提が弱い
  - 医師デモ認証情報の運用が本番向けでない

## 1. 法令・公式ガイドラインで必須確認

1. 個人情報保護法（要配慮個人情報）
- 病歴等は要配慮個人情報に該当し得るため、取得時の同意が原則必要。

2. 漏えい時の報告・通知義務
- 報告対象事態に該当する漏えい等は、PPC報告と本人通知が必要。
- 速報は速やかに、確報は30日以内（内容により60日以内）。

3. 医療・介護分野ガイダンス
- 医療・介護関係事業者向けガイダンス最新版との整合を確認する。

4. 医療広告規制
- 医師・医療機関の紹介表示は、医療広告ガイドライン・Q&Aとの整合確認が必要。

5. SaMD該当性
- AI機能が診断・治療目的に近づく場合、医療機器プログラム該当性の検討が必要。

6. 消費者法（有料化する場合）
- 通信販売表示や誇大表示規制（特商法・景表法）を満たす。

## 2. このアプリで先に塞ぐべきギャップ（優先度順）

1. 同意管理の実装（最優先）
- 初回起動時に同意画面を追加:
  - 取得情報（症状・睡眠・メモ・画像・音声）
  - 利用目的（体調管理・要約・医師共有）
  - 第三者提供の有無（医師共有リンク、外部AI API利用）
  - 撤回方法
- 同意履歴を保存:
  - `consent_version`, `agreed_at`, `scope`, `user_id`

2. 本人認証と利用者分離
- `u_123` 固定運用を廃止。
- ユーザー登録/ログイン導入（最低でもメールOTP）。
- ユーザーごとのデータ分離をサーバー側で強制。

3. 医師共有の同意粒度
- 共有リンク発行時に、共有範囲を選択可能にする:
  - 期間
  - 項目（画像含む/除外）
  - 共有先（特定医師名）
- 共有履歴の監査ログを強化（誰が/いつ/何を閲覧）。

4. デモ認証情報の本番排除
- `DOCTOR_DEMO_EMAIL` / `DOCTOR_DEMO_PASSWORD` のデフォルト値利用を禁止。
- 本番環境で未設定なら起動失敗にするガードを追加。

5. セキュリティ強化
- Cookie: `Secure + HttpOnly + SameSite` の本番強制
- レート制限のAPI全体適用（ログイン以外も）
- バックアップ/復旧手順を文書化
- 監査ログの改ざん耐性（最低でも追記専用・定期退避）

6. AI文言ガード
- UI/文言を「診断」ではなく「受診判断の補助」に固定。
- 高リスク時の受診導線を常時表示。
- おすすめ医師表示に「確定診断ではない」注記を明示。

## 3. リリース段階別のGo条件

1. デモ公開（note掲載可）
- ダミーデータのみ
- 実名/実患者情報なし
- デモである旨を明記

2. クローズドβ（実ユーザー少人数）
- 2章の1〜4を完了
- 事故対応手順（連絡先・一次対応・ログ保全）を整備

3. 一般公開
- 2章を全完了
- プライバシーポリシー/利用規約の公開
- 法務レビュー記録（承認者・日付）を残す
