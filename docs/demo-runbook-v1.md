# 体調日記 デモ実施手順（v1）

## 目的
初見の相手に3分で「記録 -> AI整理 -> 医師共有」までを見せる。

## 事前準備（開始5分前）
1. `npm start` を実行し、`http://localhost:8787/` を開く
2. Chromeを使用する（音声入力の安定性優先）
3. マイページで以下を確認
   - グラフが表示される
   - 医師共有リンク一覧が表示される

## デモ本編（3分）
1. 記録タブで症状・ゲージ・メモを入力し、`この内容で記録する`
2. `AI要約を更新` を押して、まとめ・傾向・次の一歩を表示
3. 必要なら `共有メモをコピー` を押して受診向けメモを確認
4. マイページで `共有リンク発行(24h)` を押す
5. 発行された医師ビューを開き、`/doctor-login` へ遷移したらログイン
6. 医師ビュー（`/doctor?token=...`）で内容確認
7. 医師ビューでコメント保存
8. 患者側マイページに戻り、`医師から共有されたコメント` 反映を確認

## 画像導線（追加で30秒）
1. 記録タブで画像を添付して保存（JPEG / PNG / WebP / GIF、3MB以下）
2. 同じ共有リンクで医師ビューを開き、`画像メモ` に画像が出ることを確認
3. 画像クリックで拡大モーダル表示
4. `閉じる` ボタン / 背景クリック / `Esc` で閉じる

## エラー時の表示確認（最終）
- 画像サイズ超過: `画像サイズは3MB以下にしてください` または `画像サイズが上限を超えています。3MB以下の画像を選択してください`
- 画像形式不正: `画像形式は JPEG / PNG / WebP / GIF のみ対応しています` または `画像形式が不正です。JPEG / PNG / WebP / GIF を選択してください`
- トークン不正: `共有トークンが無効です。患者さんに共有リンクの再発行を依頼してください。`
- トークン期限切れ/失効: `共有リンクの有効期限が切れたか失効しています。患者さんに再発行を依頼してください。`
- 医師未ログイン: `この操作には医師ログインが必要です。`

## 伝えるポイント
- 記録は1画面で完結
- AIは診断確定ではなく、受診判断の補助
- 医師共有は期限付きリンクで管理
- 医師コメントが患者画面に戻る

## トラブル時の即応
- 音声が入らない: 文字入力で続行（デモを止めない）
- グラフが空: マイページで`グラフ更新`
- 共有リンクが開かない: 再発行して新リンクを利用
- ログインが通らない: `doctor@example.com / doctor1234` で再入力
